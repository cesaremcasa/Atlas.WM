# Atlas.WM v4.0 Roadmap — Red Team Findings & Rebuild Plan

**Date:** 2026-07-02
**Basis:** Full adversarial code review of `main` @ `0f0a9f3` (with two empirical
re-implementations of the env dynamics), a survey of 2025–2026 world-model
literature, and an ecosystem tooling scan.
**Rule:** Phase 0 lands before any new science — several v3.x reported results
are contaminated by the bugs below and must be re-baselined.

---

## Part 1 — Red Team Findings

### Critical

**C1. The headline identifiability finding is wrong — `friction_agent` IS identifiable (R² = 0.83).**
The Block-14 exclusion of `friction_agent` rests on an "oracle MLP scoring R² < 0"
that is not committed anywhere in the repo. An independent re-implementation of
the env dynamics shows a closed-form median-of-ratios estimator (per-step velocity
decay on steps far from boxes, median-aggregated) recovers `friction_agent` with
**R² = 0.83, MAE = 0.001** under the same random policy. The agent is the only
object under continuous known excitation — it is the *most* identifiable
parameter. A least-squares variant of the same estimator scores R² = −35 because
wall/obstacle bounce steps are heavy-tailed outliers that destroy MSE-based fits —
almost certainly what broke the MLP oracle. The MODEL_CARD conclusion
("data-regime limitation, not a modeling bug") must be retracted and amended.
→ Fix in **B4/B5**.

**C2. Boxes ignore walls and obstacles and exit the grid (observed range [−62.5, +77.0] on a [0, 20] grid).**
`cruel_gridworld.py:171-187` applies wall bounce and obstacle collision to the
agent only. Consequences: `observation_space.contains(obs)` is false for most of
an episode (the physics test only checks containment at reset); the documented
"[0, 1] normalization" is actually ≈[−3.1, +3.9]; and — most importantly — boxes
drift out of the gravity interaction band (`1 < dist < 10`) and freeze outside
the grid, so after ~20–30 steps a window carries zero information about
`gravity`/`friction_box`. **The reported "oracle ceiling of 0.15–0.45 R²" is
measuring this simulation bug.** → Fix in **B1**, re-baseline in **B5**.

**C3. The `z_static_immutable` "architectural guarantee" is vacuous, and the documented mitigation was never implemented.**
The passthrough (`structured_dynamics.py:72-73`) constrains only the dynamics;
nothing constrains the encoder, whose trivially optimal solution is
`z_imm(obs) = const` (zero pred-loss on those dims, critic satisfied, decoder
ignores them). The architecture plan (line 47) prescribes a co-trained
intervention loss for exactly this reason, and the CHANGELOG marks it delivered
in Block 6 — but `grep -rn intervention src/ scripts/ tests/` returns nothing.
The tests asserting the guarantee assert a tautology of the identity op.
→ Fix in **B9**.

**C4. The adversarial action-invariance critic is structurally a no-op on this data.**
Data comes from a uniform random policy, so `a_t` is independent of `obs_t` and
`I(z_imm(obs_t); a_t) = 0` for every possible encoder — there is no action
information to remove. The critic optimum is the constant mean prediction and
the encoder gradient is an arms race around noise (the observed "adversarial
instability" that forced λ 0.01 → 0.001). Additionally the fooling-reward cap of
0.5 is mis-derived: chance-level MSE for one-hot targets is 7/64 ≈ 0.109, so the
cap rewards driving the critic ~4.6× worse than chance — i.e. embedding
*anti-information*, which is still action information. → Fix in **B9**.

**C5. In-place, sentinel-guarded normalization of the dataset files makes belief/probe results unfalsifiable.**
`train.py:63-77` divides the split `.npy` files by 20 **on disk**, guarded by a
hidden `.normalized` sentinel. `train_physics_belief.py`, `probe_physics.py`,
and `EpisodeATLASDataset` read the same files and never normalize — so the data
scale seen by the belief pipeline differs by 20× depending on script execution
order. The reported "GRU underperforms the oracle (R² 0–0.15)" is
indistinguishable from a scale mismatch. → Fix in **B2**.

### High

- **H1.** Self-predictive loss has no target network (target = detached online
  encoder output); the `lambda_latent_l2` fix is a band-aid for a known
  degeneracy class; the variance penalty contradicts the L2 term and is applied
  to the dynamics output rather than the encoder output; there is no multi-step
  rollout loss anywhere. → **B7/B8**.
- **H2.** Training is entirely unseeded (`train.py`, `train_physics_belief.py`:
  no torch/numpy seeds, unseeded shuffling DataLoader). The "determinism canary"
  tests eval-mode forward determinism, not the training pipeline. → **B3**.
- **H3.** All CI gates are inoperative: `ci.yml` disabled;
  `chaos-physics.yml` invokes `scripts/chaos_physics.py` which does not exist;
  `train-canary.yml` points at `tests/integration/test_determinism_canary.py`
  which does not exist (test lives at `tests/test_determinism_canary.py`);
  `make security` greps `torch\.load` without excluding the migration script and
  always fails. → **B4**.
- **H4.** Checkpoint signature verification is fail-open at every layer (missing
  manifest → skip; missing key → warn; unlisted file → passes; no anti-replay)
  and every call site passes `allow_unsigned=True`. → **B17**.
- **H5.** Off-by-one in episode windowing (`episode_dataset.py:79-83`): validity
  should test `cumsum[i] == cumsum[i-k+1]`; the first valid window of every
  non-first episode is dropped and a length-K episode yields 0 windows. Two
  tests assert the buggy behavior as spec. → **B2**.
- **H6.** `d_immutable` is never plumbed: configs expose
  `d_static_immutable`/`d_static_slow` but only their sum is used (split always
  `d_static // 2`), and `export_onnx.py` does not infer `d_immutable`, so
  non-default checkpoints export with the passthrough boundary at the wrong
  index. → **B3**.

### Medium (summary)

- **M1.** PhysicsBeliefEncoder is never integrated into the world model — only
  phase 1 of the RMA/VariBAD two-phase pattern exists. → **B12**.
- **M2.** One-step dynamics is ill-posed on the 6-D position-only observation:
  velocities and obstacles are unobservable, so `pred_loss` has a large
  irreducible floor. → **B6** (frame stacking).
- **M3.** Latent-probe splits sequentially over overlapping windows sharing
  physics labels (effective N ≈ #episodes) with length bias. → **B5**.
- **M4.** `generate_data.py` always overwrites raw while `split_data.py` skips
  when `.split` exists → silent stale-data training after switching physics
  modes. → **B2**.
- **M5.** `scripts/evaluate.py` computes no metric at all. → **B8**.
- **M6.** Explicit-Euler physics with force discontinuities at dist = 1.0 and
  10.0; dt-incoherent multiplicative friction. Acceptable for a toy, but should
  be documented and softened where cheap. → **B1**.
- **M7.** Dependency triple-drift between `requirements.txt` (torch 2.1.2,
  gymnasium 0.29.1), `requirements.lock` (torch 2.12.1, gymnasium 1.3.0), and
  `pyproject.toml`. → **B16**.
- **M8.** Misc: decoder never restored to `.train()` after validation; critic
  and its optimizer not checkpointed (adversarial training cannot resume);
  migrated checkpoints carry `env_hash="migrated-from-pt"` which always fails
  `strict_env=True`. → **B3/B4**.

### Low (summary)

Release tags `v3.0.0`/`v3.1.0` linked in CHANGELOG do not exist on the remote
(only `v2.0`); `pyproject.toml` says 3.0.0 vs CHANGELOG 3.1.0; README advertises
`src/atlas_wm/training/` and populated `tests/{unit,integration,physics,security}`
which are empty shells; `EntityEncoder` is permutation-invariant (not
equivariant as documented) and is dead code; several tests are decorative
(drift `< 1e6`; `err >= 0` assertions; friction tests that accept any decay);
`configs/experiment.yaml` is a stale v1 config. → **B4/B15/B18**.

### What is genuinely good (keep)

Honest CHANGELOG debugging narratives; episode-based shuffled split;
safetensors-only I/O with embedded metadata and hard `.pt` rejection; the
closed-form ridge probe (clean, leakage-free); train-stats-only physics-target
standardization; the env's seeded-RNG discipline (variable physics consumes no
extra draws when disabled, byte-identical legacy behavior, tested).

## Definition of Done (v4.0)

1. No observation-space violations (Hypothesis test in CI).
2. Bit-reproducible seeded training (training canary in CI).
3. Stable objective without the L2 crutch (EMA-vs-VICReg ablation documented).
4. `z_imm` provably informative (inter-episode variance floor; episode-identity probe).
5. `friction_agent` back in the target set with R² > 0.7 via robust features.
6. Active-exploration physics R² ≫ random-policy R² (the project's key figure).
7. Multi-step R² (K = 5) reported and trained against.
8. CI green on every PR; fail-closed or OpenSSF-signed checkpoints.
9. MuJoCo environment running through the same pipeline.
10. No claim in MODEL_CARD/README/CHANGELOG that the code does not support.
